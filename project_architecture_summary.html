🗄️ Database Schemas (Prisma Models)

Admin Account Management

Admin Model

Manages safe logins. Credentials are secured using hashed password keys (`bcrypt`) alongside safe active access JWT verification mechanisms.

id (UUID) | email (Unique) | password (Hashed) | resetOtp | otpExpiry | refresh_token

Tour & Destination Packages

Package Model

Houses individual itinerary lists, pricing structures, star ratings, Day-by-Day timelines, category mappings, inclusions, and exclusions.

id (Int PK) | title | price | duration | description | image_url | timeline (Json) | inclusions (Json) | categoryId (FK)

Bookings & Inquiries

Transactional

Holds active customer booking dates, guest information, and pending/approved reservation statuses mapped directly to packages.

Booking: id | name | email | date | status | package_id (FK)
Enquiry: id | name | email | message

⚙️ Production Stabilization Engineering

The application is 100% stable, fully compiled, and optimized. We carried out critical actions to secure this:

✓

Production Build pipeline Cleaned (`Exit code: 0`)

Discovered and corrected environment variable mismatch. Integrated `cross-env NODE_ENV=production` explicitly inside build scripts in [package.json], preventing local development parameters from crashing production workers during build-time.

✓

Deferred Hydration Guard

Implemented a deferred hydration strategy inside `ClientProviders.tsx` and `AuthInitializer.tsx`. Eager third-party initializations and Zustand browser storage checks are isolated until mounting occurs, fully preventing Next.js static site generation (SSG) worker errors.

✓

Silent Network Failures Management

Streamlined Axios response interceptors inside `axiosInstance.ts` so that temporary GET load errors on public components (like categories loaded in navigation menus) fail silently instead of triggering intrusive modal toast warnings. Active write updates inside the Admin Dashboard remain protected.
